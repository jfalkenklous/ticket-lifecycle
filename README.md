<h1>osTicket - Ticket Lifecycle: Intake Through Resolution</h1>
This tutorial outlines the lifecycle of a ticket from intake to resolution within the open-source help desk ticketing system osTicket.<br />

<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10</b> (22H2)

<h2>Ticket Lifecycle Stages</h2>

- Intake
- Assignment and Communication
- Working the Issue
- Resolution

<h2>Lifecycle Stages</h2>

<p>
Navigate to where an end-user can open a new ticket (http://localhost/osTicket/). Select 'Open a New Ticket'. Fill in the information for the ticket, similar to what is shown in the image above, then hit 'Create Ticket'.

<p>
Login to the Agent Panel as John. Now we are able to see the ticket created by Karen. Click into the ticket by clicking the ticket number or by clicking the subject of the ticket. 
</p>

<img width="1140" height="903" alt="image" src="https://github.com/user-attachments/assets/40940f8d-8159-4c84-b8e7-95ef234a3fda" />

<p>
Update the SLA Plan to 'Sev-A'. Update the Help Topic to 'Report a Problem/Business Critical Outage'. Update the Assigned To field to 'Jane Doe'. Log out as John and now login as Jane. 
</p>

Click into the open ticket. Add a response, similar to what was typed in the image above and then hit 'Post Reply'.
